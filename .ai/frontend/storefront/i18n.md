# Storefront Internationalisation (i18n)

Updated: 2026-04-09
Owner: @ivans

## Stack

- `i18next` + `react-i18next` — translation engine
- `i18next-browser-languagedetector` — auto-detects language from `localStorage` → browser → `htmlTag`
- `languageSlice` (Redux) — tracks current language and text direction for the rest of the app
- Supported locales: **`en`** (primary), **`bg`** (secondary)
- RTL-ready: `languageSlice` stores `direction: 'ltr' | 'rtl'`; adding an RTL locale requires only a `dir` entry in the languages config

## Files

| File | Role |
|---|---|
| `src/shared/i18n/index.ts` | i18next initialisation; exports `changeLanguage`, `getLanguageDirection`, `languages`, `defaultLanguage` |
| `src/shared/i18n/locales/en.json` | English translations (source of truth for key names) |
| `src/shared/i18n/locales/bg.json` | Bulgarian translations (must mirror `en.json` key structure) |
| `src/shared/i18n/locales/index.ts` | Barrel — exports `resources`, `languages`, `defaultLanguage` |
| `src/shared/i18n/languageSlice.ts` | Redux slice: `currentLanguage`, `direction`, `initialized`; `setLanguage` async thunk |
| `src/shared/hooks/useTranslation.ts` | Wrapper hook — use this, not raw `useTranslation` from `react-i18next` |
| `src/app/hooks/useLanguageMetaSync.ts` | Syncs `<html lang>` and `<html dir>` on language change |
| `src/app/LanguageSwitcher/` | UI component — dispatches `setLanguage` thunk |

## Namespace structure

All translations live in a single `"translation"` namespace (i18next default). Keys are grouped by domain at the top level.

| Namespace key | Domain |
|---|---|
| `common` | Shared UI labels: Save, Cancel, Loading, Error, etc. |
| `nav` | Navigation labels: Home, Products, Cart, Sign In, etc. |
| `theme` | Theme switcher: Light, Dark, System |
| `announcement` | Announcement bar content |
| `home` | Home page section headings and CTAs |
| `products` | Product listing, filters, sort, grid |
| `productDetail` | Product detail page: title, reviews, add to cart |
| `cart` | Cart page and cart item labels |
| `checkout` | Checkout form, payment, order confirmation |
| `account` | Account/profile section headings |
| `orders` | Order history and order detail |
| `wishlist` | Wishlist page |
| `auth` | Login, register forms |
| `forgotPassword` | Forgot password flow |
| `resetPassword` | Reset password flow |
| `profile` | Profile form labels |
| `trackOrder` | Track order page |
| `trustSignals` | Trust badges, shipping info |
| `validation` | Client-side validation error messages |
| `errors` | API and generic error messages |
| `footer` | Footer links and labels |
| `staticPages` | About, Blog, Careers, etc. content |

## How to use translations in components

**Always import via the wrapper hook, not `react-i18next` directly:**

```tsx
// ✅ correct
import { useTranslation } from '@/shared/hooks/useTranslation';

export function AddToCartButton() {
  const { t } = useTranslation();
  return <button>{t('cart.addToCart')}</button>;
}

// ❌ avoid — bypasses the project wrapper
import { useTranslation } from 'react-i18next';
```

**Interpolation:**
```tsx
// en.json: "viewAllResults": "View all results for \"{{query}}\""
t('common.viewAllResults', { query: searchTerm })
```

**`useTranslation` hook returns:**
```ts
{
  t,                  // translation function
  i18n,               // i18next instance
  currentLanguage,    // 'en' | 'bg'
  changeLanguage,     // async (lang: string) => void
  isEnglish,          // boolean
  isBulgarian,        // boolean
}
```

## How to change the active language

**From a component or hook — dispatch the thunk:**
```tsx
import { useAppDispatch } from '@/shared/lib/store';
import { setLanguage } from '@/shared/i18n/languageSlice';

function LanguageSwitcher() {
  const dispatch = useAppDispatch();
  return (
    <button onClick={() => dispatch(setLanguage('bg'))}>БГ</button>
  );
}
```

`setLanguage` calls `changeLanguage()` from `i18n/index.ts`, which calls `i18n.changeLanguage()`, saves to `localStorage`, and updates `document.documentElement.lang/dir`. The Redux slice then updates `currentLanguage` and `direction`.

Do NOT call `i18n.changeLanguage()` directly from a component — use the thunk so the Redux state stays in sync.

## Adding a new translation key

1. Add the key to `src/shared/i18n/locales/en.json` under the correct namespace.
2. Add the matching key to `src/shared/i18n/locales/bg.json` with a Bulgarian translation.
   - If the Bulgarian translation is not ready yet, copy the English string temporarily.
   - Never leave a key missing from `bg.json` — i18next falls back to the key string, not the English value.
3. Use `t('namespace.keyName')` in the component.

**Key naming rules:**
- `camelCase` — no dots within a key name (dots are namespace separators in i18next)
- Use the existing namespace if the key belongs to a known domain
- If no namespace fits, add to `common` and refactor to a new namespace only when three or more keys belong to the same domain

## Adding a new language

1. Create `src/shared/i18n/locales/{code}.json` — must mirror the structure of `en.json`
2. Add the language to the `languages` array in `src/shared/i18n/locales/index.ts`:
   ```ts
   { code: 'fr', label: 'Français', dir: 'ltr' }
   ```
3. Update `LanguageSwitcher` UI to show the new option
4. If the language is RTL (e.g. Arabic), set `dir: 'rtl'` — the `languageSlice` and `useLanguageMetaSync` handle the rest

## Language detection order

On initial load, i18next detects the language in this order:

1. `localStorage` key `"language"` — set by `setLanguage` thunk
2. Browser `navigator.language`
3. `<html lang>` attribute

Once detected, the language is persisted to `localStorage` on every `setLanguage` call.

## Common mistakes

- **Using raw `react-i18next` hook** — use `@/shared/hooks/useTranslation` instead
- **Calling `i18n.changeLanguage()` directly** — use the `setLanguage` thunk to keep Redux state in sync
- **Key missing from `bg.json`** — i18next shows the key string, not a fallback translation
- **Adding keys directly to `common` without checking existing namespaces** — always check the namespace table above first
- **Hardcoding English strings in components** — even strings that look static must go through `t()` so they change on language switch
